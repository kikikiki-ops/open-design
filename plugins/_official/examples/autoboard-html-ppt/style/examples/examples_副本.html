R1 卡片高度控制
  • ✕禁止 height:100%、flex:1 强制拉伸至底部
  • ✓卡片优先 height: auto
  • ✕正文 <120字时，卡片高度不得超过画布 55%
  • ✓有效内容占卡片高度 ≥ 55%
  • ✓留白分布在模块外部而非卡片内部
  • ✓等高基准 = 内容最多的卡片，非画布剩余高度
R2 页面填充与留白控制
视觉占比要求
标题区10%–18%
主体内容区45%–65%
辅助视觉区15%–30%
连续留白上限≤ 25%
有效覆盖面积55%–75%
内容不足时优先：放大信息 → 升级主视觉 → 改为流程结构 → 增加图形 → 最后才调整尺寸
R3 内容密度判断
高密度
紧凑卡片、分栏、表格
中密度
卡片 + 数据图形组合
低密度
少量大模块、主视觉数据、流程图
✕ 禁止所有密度都套用三等分卡片布局
R4 同组卡片对齐协议
内容多少不同，外框必须等高对齐
  • ✓外框对齐优先于内容对齐
  • ✓父级容器统一控制（Grid/Flex）
  • ✕禁止逐张绝对定位、独立 margin
  • ✓生成后用 getBoundingClientRect 验证（误差 ≤2px）
R5 异构模块对齐规则
4 种对齐锚点
①
外边界锚点：同层级模块共享顶/左基线
②
行轨道锚点：对应列表共享行高行距
③
中心线锚点：箭头对准连接对象中心
④
语义关系优先：对应→逐行，因果→连线，总结→共顶
✕ 禁止补偿式定位（margin/translate/absolute 修复对齐）
R6 同一内容带对齐协议
所有一级模块顶底对齐，高度统一为 H
cardsAreaHeight = H - titleH - gap
singleCardH = (cardsArea - cardGap × (n-1)) / n
