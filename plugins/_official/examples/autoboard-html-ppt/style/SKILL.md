# 超宽大屏商务增长风格 PPT Skill

## 适用场景

当用户需要基于「超宽会场大屏 / 品牌营销大会 / 商业化业务汇报 / 数据增长发布 / 案例复盘」生成或改写 PPT 页面时，使用本 Skill。

本 Skill 固定适配以下画布：

- 物理尺寸：11m × 3m
- 像素分辨率：3696 × 1008 px
- 画面比例：11:3（约 3.67:1）
- 输出方向：超宽横屏

视觉基准来自用户提供的快手品牌营销大屏样稿，整体方向为：浅暖底、柔光流线、香槟金商务感、青绿色轻科技感、深青黑标题、白色圆角卡片、低信息密度、中心聚焦式表达。

---

## 设计目标

生成页面时，不要把普通 16:9 PPT 横向拉伸，而要按照「会场超宽主屏」设计。

当总控已选择开发者大会模板时，还必须读取：

```text
../orchestrator/rules/template_library.md
```

模板定义页面结构和语义槽位；本 Style Skill 只负责将其绑定到视觉 token、
背景、独立 Logo 和组件样式。不得把 2.67:1 参考稿横向拉伸到 3.67:1：
超宽空间只能由轨道、模块间距、侧栏或图表区域吸收，卡片、图片、截图和圆形图形
必须保持自身比例。

核心目标：

1. 远距离一眼可读。
2. 一页只讲一个核心结论。
3. 中间承载主信息，左右用于氛围延展。
4. 通过白色圆角卡片承载结构化内容。
5. 用香槟金表达商业成果，用青绿色表达智能策略 / 数据增长。
6. 画面保持高级、明亮、克制、商务、轻科技。

## 工作流程

### Step 2：确定页面结构

根据页面类型选择对应结构：

#### 封面页

结构：

- 顶部品牌区
- 中心大标题
- 可选副标题
- 底部柔光流线

适合：会议主题、品牌主题、发布会主视觉。

#### 章节过渡页

结构：

- 顶部品牌区
- 中心章节标题
- 极少量辅助说明
- 底部轻氛围背景

适合：章节切换、议题转场。

#### 案例成果页

结构：

- 顶部品牌区
- 居中主标题
- 中部三段式内容：案例背景 / 方法策略 / 成果结果
- 右侧或中部突出关键数字

适合：品牌案例、项目复盘、业务成果。

#### 数据结论页

结构：

- 居中主标题
- 主图表或核心逻辑图
- 2~4 个关键数字
- 少量辅助说明

适合：趋势洞察、相关性分析、增长结果、人群价值说明。

#### 策略分析页

结构：

- 居中主标题
- 中心问题 / 目标
- 左右展开「问题 - 解法 - 结果」
- 可使用标签、箭头、卡片映射

适合：策略升级、投放解法、资源组合、方法论表达。

---

### Step 3：应用固定画布与安全区

> **安全区规则唯一来源**：`style/examples/example_prompt.md §1.3` 及 `R20 一`。
> 速查：画布 3696 × 1008 px；左右各 220px、上下各 90px；可用内容区 3256 × 828px；`.safe-zone { left:220px; right:220px; top:90px; bottom:90px; }`；Logo 底边约 99px，内容顶部需留间隙（多列页 40px，单列页 32px）。

核心信息必须放在安全区内。左右边缘主要做背景氛围，不放重要文字。

---

### Step 4：应用视觉风格

必须使用以下视觉原则：

- 背景为浅暖白 / 米白。
- 底部或左右两侧有低透明度青绿色 + 香槟金柔光流线。
- 主标题居中，使用深青黑。
- 主体内容使用白色大圆角卡片。
- 卡片边框极细，阴影极轻。
- 关键数字用香槟金或深青黑放大展示。
- 图表用青绿色为主，香槟金为辅。
- 信息密度低，远距离可读。

详细 token 见 `style.md`。

---

### Step 5：控制内容密度

单页建议上限：

- 1 个主标题
- 1 个核心结论
- 2~4 个主要模块
- 1 张主要图表
- 2~4 个大数字
- 5 个以内辅助标签
- 30~80 字正文说明

内容过多时，处理顺序：

1. 检查内容账本与关系绑定
2. 重新分组并选择更适配的布局变体
3. 拆成连续页面，并保留来源页与叙事顺序
4. 仅弱化用户明确标记为辅助的内容

严禁：缩小字号硬塞、压缩边距、密集表格、满屏小字。

### 如果输出 HTML 型 PPT 页面

#### ⚠️ 背景图与 Logo 资产必须来自 `assets/`（P0 硬约束）

在输出任何 HTML 之前，必须已完成 orchestrator K-0 步骤，确保以下 4 个文件真实存在：

```
assets/bg-cover.svg      ← 封面背景
assets/bg-content.svg    ← 内容页背景
assets/bg-closing.svg    ← 封底背景
assets/logo.svg          ← 快手联盟 Logo
```

**这 4 个文件由 K-0 从 `.od-skills/autoboard-html-ppt-<hash>/style/assets/` 复制而来。**

每张 slide 的写法（封面示例）：
```html
<section class="ppt-slide active" data-page-role="cover">
  <!-- ① 背景图：第一个子元素 -->
  <img class="bg-img" src="assets/bg-cover.svg" alt="" aria-hidden="true" />
  <!-- ② 独立 Logo：紧跟背景图 -->
  <img class="fixed-brand-logo" src="assets/logo.svg" alt="快手联盟" data-asset-role="fixed-brand-logo" />
  <!-- ③ 内容安全区 -->
  <div class="safe-zone">…</div>
</section>
```

**严禁：**
- 用 `source-media/*.png/jpg` 作为背景 — 这是用户 PPTX 里的图片，不是品牌背景
- 用任何 `source-media/` 文件作为 Logo — logo 必须是 `assets/logo.svg`
- 引用绝对路径 `/Users/...` 或 `.od-skills/.../assets/...`
- 省略 `<img class="bg-img">` — 不得用 CSS `background-image` 替代
- 省略 `<img class="fixed-brand-logo">` — Logo 在每页都必须独立渲染

如果 `assets/` 目录不存在或上述文件缺失，**禁止继续输出 HTML**，先报告错误。

---

必须：

- `.ppt-slide` 容器使用固定尺寸：`width: 3696px; height: 1008px;`（统一使用 `.ppt-slide`，不得使用 `.slide`）
- `html, body, .deck-stage` 使用 `width:100vw; height:100vh; overflow:hidden;`
- **必须加入等比自适应缩放脚本**（见下方模板），确保任意屏幕尺寸下 3696×1008 都能完整等比居中显示
- 使用 `position:absolute` 绝对布局，`transform-origin: 0 0`
- 所有核心内容位于安全区内（左右各 220px，上下各 90px）
- 背景流线为装饰层，不影响内容层
- 字号：遵循 `style/examples/example_prompt.md §3.2`（正文不得小于 24px）
- 不要出现页面滚动
- 不要出现内容溢出
- 所有正式内容保留 `data-source-id`，页面保留 slide index、角色和背景变体

#### 必须包含的自适应缩放脚本模板

**每一个生成的 HTML PPT 都必须在 `</body>` 前包含以下脚本**（可根据是否有导航栏调整）：

```html
<script>
(function () {
  var SLIDE_W = 3696, SLIDE_H = 1008;
  var slides = Array.from(document.querySelectorAll('.ppt-slide'));  // 统一选择器：.ppt-slide

  function scaleSlides() {
    var navEl = document.getElementById('nav-hud');
    var navH  = navEl ? navEl.offsetHeight : 0;
    var vw    = window.innerWidth;
    var vh    = window.innerHeight - navH;
    var scale = Math.min(vw / SLIDE_W, vh / SLIDE_H);
    var ox    = (vw - SLIDE_W * scale) / 2;
    var oy    = (vh - SLIDE_H * scale) / 2;
    slides.forEach(function (s) {
      s.style.transform       = 'scale(' + scale + ')';
      s.style.transformOrigin = '0 0';
      s.style.left            = ox + 'px';
      s.style.top             = oy + 'px';
    });
  }

  document.addEventListener('DOMContentLoaded', scaleSlides);
  window.addEventListener('load',   scaleSlides);
  window.addEventListener('resize', scaleSlides);
})();
</script>
```

**不允许**使用 `width:100vw; height:100vh` 直接拉伸 `.ppt-slide` 容器，这会破坏 3.67:1 超宽比例。

> ⚠️ **统一说明**：本 Skill 全局使用 `.ppt-slide` 作为每张幻灯片的根类名，与 `html_output_contract.md` §1.1 保持一致。禁止同时使用 `.slide`、`.slideshow`、`.ppt-page` 等别名。

## 卡片高度控制（硬约束）

生成内容卡片时必须遵守：

- 内容较少时不得将卡片强制拉伸到画布底部。
- 卡片默认使用 `height: auto`；禁止默认使用 `height: 100%`、`flex: 1` 或过大的 `min-height`。
- 单张卡片正文少于 120 字时，卡片高度不得超过画布高度的 55%（554px）。
- 卡片有效内容占卡片高度的比例不得低于 55%。
- 剩余留白放在模块外部的布局间距中，不得集中留在卡片内部。
- 先识别横向、纵向或矩阵语义卡片组，再用共同父容器统一控制位置、尺寸、间距和对齐；禁止逐张定位。
- 横向组必须顶边、底边对齐且外框等高；纵向组必须左、右边缘对齐且外框等宽；矩阵组各行等高、各列等宽，行列间距一致。
- 内容差异只能通过内部弹性空间、固定底部结论区或精简文案处理；外框对齐优先于内部内容对齐，不得移动整张卡片。
- 主次卡、瀑布流、时间轴或刻意非对称构图可例外，但必须在页面计划中说明设计意图。

## 超宽画布中的卡片宽度、文本行长与图片裁切（硬约束）

### 一、总原则

超宽画布只代表页面可使用的横向空间更大，不代表单个内容模块也应无限变宽。页面生成前必须先建立"内容最大宽度"约束，不得将文本卡片、图片卡片或内容模块直接按画布剩余宽度无限拉伸。

### 二、文本卡片宽度规则

- 普通说明型文本卡片：宽度控制在画布宽度的 **22%–32%**
- 信息较多的文本卡片：宽度不超过画布宽度的 **38%**
- 单栏长文本区域：宽度不超过主体内容区的 **45%**
- 即使横向空间充足，也不得让单段正文横跨过长距离
- 文本卡片必须使用 `width: 100%` 配合 `max-width`，禁止只用无上限的 `1fr` 拉伸

```css
.text-card {
  width: 100%;
  max-width: var(--text-card-max-width); /* 建议值：800px–1200px */
}
```

超宽画布中优先使用多列布局、左文右图、主卡 + 辅卡、短文本 + 数据视觉，而不是将少量文字放进一张极宽卡片。

### 三、正文行长规则

- 中文正文单行建议控制在约 **22–36 个汉字**
- 大屏演示场景可适度增加，但单行不宜超过约 **40 个汉字**
- 说明型小字不得在超宽卡片中形成一整条横向长句
- 文字过少时应缩窄文本区，不得让文字贴着卡片左侧、右侧留下大面积空白
- 文字过多时应拆分层级、分栏或提炼内容，不得持续扩大卡片宽度

```css
.card-copy {
  max-width: 34ch;   /* 中文正文硬约束 */
  line-height: 1.5;
}
```

标题、正文、标签可以拥有不同的最大宽度，正文不必跟随卡片整体宽度铺满。

### 四、图片卡片宽度与比例规则

每个图片容器必须具有明确宽度、明确高度或 `aspect-ratio`、`overflow: hidden`、统一圆角和统一裁切方式。

```css
.image-card {
  width: 100%;
  height: 100%;
  overflow: hidden;
  border-radius: var(--card-radius);
}

.image-card img {
  width: 100%;
  height: 100%;
  object-fit: cover;
  object-position: center;
  display: block;
}
```

**禁止默认使用：**
- `object-fit: contain`
- 图片按原始比例缩小后悬浮在容器中
- 图片只占容器中间一小部分
- 图片上下或左右出现无意义空白
- 通过拉伸图片宽高填满容器

### 五、图片必须优先使用 cover 裁切

当图片作为卡片主视觉、背景图、案例图、场景图或氛围图时，默认使用 `object-fit: cover`。

裁切时必须根据图片主体位置设置 `object-position`，不得机械地对所有图片使用完全相同的中心裁切：

| 主体位置 | object-position |
|----------|----------------|
| 居中 | `center center` |
| 人物偏左 | `35% center` |
| 人物偏右 | `65% center` |
| 重点位于上方 | `center 30%` |
| 重点位于下方 | `center 70%` |

AI 必须先判断主体位置，再确定裁切焦点。

### 六、图片比例选择规则

图片显示比例服从页面结构优先，同时尊重图片构图完整性——**比例是构图建议，不要求裁掉主体来强行匹配**。

| 用途 | 建议比例 | 说明 |
|------|---------|------|
| 横向主图 | **16:10、4:3** 或适合页面栏宽的自然比例 | 优先保留图片主体，不得为凑比例裁掉人脸或主体 |
| 横向场景图 | 16:9、3:2 或 2:1 | 大屏氛围、产品环境、活动现场 |
| 品牌氛围图 | 4:3 或 3:2 | 商务发布场景首选 |
| 人物肖像 | **3:4**（竖图） | 单人或小组发言者图；多图保持同高 |
| 多图组 | 各图同高，或共享基线 | 禁止一张横图一张竖图混排而不统一容器 |
| 超宽背景图 | 2.4:1 至 3.5:1 | 封面/封底全屏背景 |
| 小型缩略图 | 1:1 | 头像、图标、徽章 |

**同组图片规则**：
- 多图组保持**同高**或共享同一基线；
- 禁止一张横图一张竖图混排而容器未统一；
- 裁切方式统一（均用 `object-fit: cover`），边界可对齐；
- 图片内容不得漂浮在容器中间（禁止 `object-fit: contain` 导致四周留白）。

### 六·一、生成图禁止事项

**生成图（AI 生成图片）不得自带以下元素**，这些均由页面 HTML 单独排版，不得内嵌在图片内：

- PPT 标题文字 / 页面标题；
- 页眉 / 页脚文字；
- 页码 / 幻灯片编号；
- 图片说明（caption）；
- 装饰边框 / 外框线条；
- 任何叠加文字（overlay text）或水印。

生成图应为**纯视觉内容**，文字说明由 HTML `<p>`、`<figcaption>` 或标题元素单独排版。
### 七、图文组合卡片规则

当一个模块同时包含图片和文本时，不得让图片区和文本区都无限拉宽。

| 类型 | 图片占比 | 文本占比 |
|------|---------|---------|
| 图片主导型 | 55%–70% | 30%–45% |
| 信息主导型 | 35%–45% | 55%–65% |
| 均衡型 | ~50% | ~50% |

超宽画布中不建议出现单侧超过模块宽度 75% 的普通内容区。

```css
.media-card {
  display: grid;
  grid-template-columns:
    minmax(0, var(--image-max-width))
    minmax(0, var(--text-max-width));
  gap: var(--content-gap);
}
```

图片区和文本区都必须有独立的最大宽度约束，不能仅依靠 `1fr 1fr` 无限扩展。

### 八、低内容量处理规则

当卡片内容较少时，禁止通过增加卡片宽度、拉长正文行长、增加无意义内边距、将小图片等比缩放后放大容器、增加无信息价值空白区域来填满超宽空间。

应按以下顺序处理：
1. 缩小单个卡片最大宽度
2. 调整模块为紧凑组合
3. 增加列间距或组间距
4. 将关键图片放大并使用 cover 裁切
5. 将数据、关键词或结论升级为视觉模块
6. 使用不对称构图分配剩余空间
7. 必要时增加与内容相关的图表或视觉表达

### 九、卡片组最大宽度规则

同一行多个卡片时，应同时约束单卡最大宽度、卡片组总宽度、卡片间距和页面左右安全边距。

```css
.card-group {
  width: min(100%, var(--group-max-width));
  margin-inline: auto;
  display: grid;
  gap: var(--card-gap);
}
```

超宽画布中，内容组可居中，也可按页面构图偏左或偏右，但必须有明确最大宽度，**禁止主体内容从左安全区一直连续铺到右安全区**。

## 异构模块对齐（硬约束）

- 列表、卡片、箭头、数据指标、图表和总结模块共存时，先建立外边界、行轨道和中心线锚点；不得将所有模块强制等宽或等高。
- 对应列表共享行数、行高、行间距及每项的顶部、底部和中心线；箭头和连接线必须对准所连接对象中心。
- 不同类型模块可不同宽高，但至少共享一个外边界锚点和一个内容锚点；总结模块至少与主体模块共享一个外边界。
- 有对应关系优先逐行对齐，有因果关系优先用连接线和中心线对齐，有总结关系优先通过共同顶部和内容中心对齐。
- 禁止用逐模块 `margin`、`top`、`left` 或 `translate` 补偿，必须回到共同父级 Grid、Flex、行轨道和对齐变量修正。

### 同一内容带异构模块（硬约束）

- 同一横向内容带内的一级模块必须由共同父级定义统一高度 `H`，通常为画布高度的 35%～55%，并顶边、底边对齐。
- “标题条 + 卡片列表”模块以统一标题高度和间距分割出卡片区；卡片区通过 Grid 等分剩余高度。单张数据、成果和图表模块外框同样为 `H`，内容可居中但不得把外框做得更高。
- 一级模块、同组卡片和逐项对应行的 DOM 几何边界误差必须小于等于 2px；修复只能调整共同父级内容带和内部轨道，不得局部位移补偿。

## 形状、卡片与装饰大版式动作（硬约束）

### 允许的基础形状
- 圆角矩形（`border-radius`）
- 严格正圆（`aspect-ratio:1/1`，不得因内容变椭圆）
- 直角小方点（2px–8px，常见于点阵、步骤序号、子弹点）
- 点阵（均匀小点矩阵，低透明度背景装饰）
- 规则网格（等间距横/纵线，辅助结构感）

### 大尺度几何动作（每页限一个，不得叠加）

每个页面允许选用**一种**大尺度几何手法作为视觉锚点：

| 手法 | 适用场景 | 约束 |
|------|---------|------|
| **通高色列** | 标题页、章节页、侧边强调 | 宽度 ≤ 画布宽度 3%；位于内容层后方 |
| **巨型数字** | 核心指标页、成果页、覆盖类数据页 | `opacity: 0.06–0.20`；不遮文字 |
| **粗横杠** | 分隔区块、强调标题区 | 高度 ≤ 画布高度 3%；渐变或纯色均可 |
| **单条斜线** | 区分区块、引导视线 | `stroke-width: 1px–3px`；不得粗实线 |

**禁止**：
- 同页叠加两种及以上大尺度几何动作；
- 大尺度元素遮挡文字、数字或卡片主体；
- 引入品牌色板以外的颜色；
- 将巨型数字 `opacity` 设为 > 0.20（抢占内容层）；
- 将通高色列或粗横杠的尺寸超过画布的 3%。

### 反定义：以下不是"大尺度几何动作"

下列元素**不属于**允许使用的大尺度几何动作，不得将其混入上表的一次配额中：

| ❌ 误用 | 原因 |
|--------|------|
| 多个圆形气泡平铺背景 | 装饰泛滥，无结构锚点 |
| 任意色块拼贴 | 非几何约束，制造视觉噪声 |
| 多条平行装饰线 | 超出"单条"约束 |
| 全页渐变蒙层叠加在背景图上 | 属于背景处理，不属于几何动作 |
| 随机散落的点 / 不规则点阵 | 须为均匀规则，否则为噪声 |

### 图片与版式反定义（硬约束）

❌ **禁止为填空白加入无关图库照片**：图片必须与当前页面内容直接相关；不得以"丰富视觉""增加层次感"为由插入与话题无关的通用图库图片（如无意义的握手、键盘、人群背影等）。

❌ **禁止每页重复相同的"左图片右三卡"结构**：同一 deck 中不得让所有内容页套用同一种图文排列模板。不同页面应根据内容选择不同分栏比例（参见 §5.6），产生视觉节奏差异。

## 禁止项

生成时避免：

- 赛博蓝紫
- 高饱和霓虹光
- 强发光边缘
- 复杂 HUD 科技线框
- 大面积深色底
- 过厚投影
- 过多渐变块
- 拟物按钮
- 花哨字体
- 卡通插画感
- 杂乱粒子
- 大段长文
- 小字号密集表格
- 平均铺满整条长屏的碎信息

```text

[ ] 图表是否简洁清晰

[ ] 是否满足远距离观看可读性

[ ] 文本区是否铺满了不必要的卡片宽度（若是，应缩窄或拆分）
```

## 一句话风格提示词

采用超宽大屏商务增长风格，画布为 3696×1008，适配 11m×3m 会场主屏。背景为浅暖白和米白色，底部与两侧加入低透明度青绿色与香槟金柔和流线。主标题居中，信息重心集中在中间区域，左右作为氛围延展。主体内容使用白色大圆角卡片承载，边框细、阴影极轻。颜色以深青黑文字、香槟金重点和青绿色点缀为主。突出大数字、关键结论与简洁图表，信息密度低，远距离易读。避免赛博炫光、复杂科技 HUD、厚重阴影、高饱和渐变、密集小字和拥挤排版。

## 固定背景图系统

本风格 Skill 支持固定封面、内容页、封尾三类背景图。背景图资产由本 Skill 管理，详见 `background_system.md` 与 `asset_manifest.json`。

页面背景枚举：

```text
cover：封面页背景
content：正文内容页背景
closing：封尾页背景
```

分工规则：

```text
风格 Skill：定义背景图资产、视觉强度和使用规则
总控 Skill：判断页面角色并传递 backgroundVariant
组件 Skill：根据 backgroundVariant 渲染背景图
```

背景图不得承载必须可编辑的信息。主标题、正文、关键数字、单位、图表标签、Logo、二维码等应作为独立 HTML / 组件呈现。

## 固定品牌信息禁用规则

背景图不承载品牌标识，因此本风格 Skill 不应额外生成任何常驻品牌文字或顶部小标签；Logo 仅通过独立组件渲染。

默认禁止在页面中自动叠加：

- 左上角第二个“快手联盟”
- 顶部常驻品牌文字
- 顶部常驻业务线文字
- 固定角标
- 未来自原始 PPT 正式内容的品牌副标识

如果原始 PPT 页面本身明确提供了品牌文字、会议名称、年份、演讲人信息等，才允许作为可编辑 HTML 文本输出。否则不得凭风格 Skill 自动补充。

## 背景无 Logo 与独立 Logo 组件规则

当前版本采用“背景图不带 Logo，Logo 独立组件渲染”的方案。

必须遵守：

- 背景图只承载流线、光影、色块、纹理与空间氛围
- Logo 不得烘焙进背景图
- Logo 必须通过独立组件渲染
- 封面 / 内容 / 封尾均显示 Logo
- Logo 资产优先使用 SVG
- 不得在 Logo 下方额外生成第二行“快手联盟”文字

Logo SVG 源地址：

```text
https://p5-ad.adkwai.com/udata/pkg/ks-ad-fe/md-tools/quick-cut/6689:5117.b90a9c9d53932e9b.svg
```

默认组件能力：`FixedBrandLogo`。
