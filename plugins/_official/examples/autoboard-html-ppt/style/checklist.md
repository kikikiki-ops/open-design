# 质量检查清单
# 超宽大屏商务增长风格

## 1. 画布检查

```text
[ ] 画布是否为 3696 × 1008 px
[ ] 是否适配 11m × 3m 超宽大屏
[ ] 是否是 11:3 超宽构图，而不是 16:9 横向拉伸
[ ] 是否无滚动、无裁切、无内容溢出
```

## 2. 安全区检查

```text
[ ] 核心信息是否位于 X: 220~3476 px 范围内
[ ] 核心信息是否位于 Y: 90~918 px 范围内
[ ] 左右边缘是否没有重要文字
[ ] 顶部和底部是否预留了显示安全空间
```

## 3. 版式检查

```text
[ ] 页面是否中心聚焦
[ ] 左右两侧是否主要用于氛围延展
[ ] 主体模块是否横向展开
[ ] 页面是否没有被平均铺满
[ ] 模块之间间距是否均匀
[ ] 视觉重心是否清晰
```

## 4. 字体检查

> 字号标准唯一来源：`style/examples/example_prompt.md §3`

```text
[ ] 字体 / 字号 / 字重是否符合 example_prompt.md §3.1–§3.4
[ ] 正文字号是否不小于 24px（最小值，见 §3.2 --font-body-xs）
[ ] 主标题是否不超过 2 行，line-height:1
[ ] 是否没有使用花哨字体 / 手写体 / 卡通体
```

## 5. 色彩检查

```text
[ ] 背景是否为浅暖白 / 米白
[ ] 主文字是否为深青黑
[ ] 重点结果是否使用香槟金
[ ] 策略 / 数据 / 智能信息是否使用青绿色
[ ] 是否没有出现赛博蓝紫、高饱和霓虹色
```

## 6. 卡片检查

```text
[ ] 主体是否使用白色大圆角卡片
[ ] 卡片圆角是否约 22 px
[ ] 卡片边框是否极细
[ ] 卡片阴影是否极轻
[ ] 卡片内边距是否充足
[ ] 是否没有厚重投影和拟物按钮感
[ ] 内容较少时，卡片是否未被强制拉伸到画布底部
[ ] 卡片是否默认使用 height: auto，且未默认使用 height: 100%、flex: 1 或过大的 min-height
[ ] 正文少于 120 字的卡片是否不超过画布高度的 55%（554px）
[ ] 卡片有效内容占卡片高度的比例是否不低于 55%
[ ] 页面剩余留白是否分布在模块外部，而非集中在卡片内部
[ ] 是否先识别横向、纵向或矩阵语义卡片组，并由共同父容器统一控制
[ ] 横向组是否顶边、底边对齐、外框等高且水平间距一致
[ ] 纵向组是否左、右边缘对齐、外框等宽且垂直间距一致
[ ] 矩阵组是否各行等高、各列等宽，且行列间距一致
[ ] 同组卡片是否未逐张设置 top、left、margin、translate 或不同的尺寸计算方式
[ ] 内容量不一致时，是否统一外框、顶部对齐内容、用内部弹性空间吸收差异，并固定底部结论或操作区
[ ] 同组边界误差是否不超过 2px，且无单卡片横向或纵向偏移
```

## 6.1 同一内容带异构模块检查

```text
[ ] 同一横向内容带是否由共同父级定义了统一高度 H，且 H 约为画布高度的 35%～55%
[ ] 同带所有一级模块是否 top、bottom、height 完全对齐，误差均不超过 2px
[ ] 内容带 H 是否未由异常高单卡、页面剩余高度、100vh、独立 min-height 或 flex: 1 决定
[ ] “标题条 + 多卡片”模块是否使用统一标题高度、统一间距和内部 Grid 等分卡片区
[ ] 有逐项对应关系的两组卡片是否共享行轨道，逐项 top、bottom、height 误差均不超过 2px
[ ] 单体数据、成果和图表模块是否与同带模块等高，且只通过内部内容层级而非空容器制造强调
[ ] 箭头、流程符号和过渡文字是否未参与一级模块高度计算，并对准关联对象内容中心或对应行中心
[ ] 是否未使用单独 margin、top、bottom、left、right、translate、独立 height/min-height、绝对定位或无意义 padding 补偿
[ ] 是否使用 getBoundingClientRect() 读取一级模块和对应卡片的几何数据，而非只凭肉眼验收
```

## 6.2 异构模块对齐检查

```text
[ ] 列表、卡片、箭头、指标、图表和总结模块是否已建立外边界、行轨道或中心线锚点
[ ] 有逐项对应关系的列表是否共享行数、行高、行间距、顶部、底部和中心线
[ ] 箭头、连接线和转折符是否对准连接对象中心，而非页面中心
[ ] 总结模块是否至少与主体模块共享一个外边界；不同类型模块是否还共享一个内容锚点
[ ] 是否未通过单独 margin、top、left 或 translate 伪造对齐
[ ] 模块间是否形成明确阅读路径，且不存在视觉接近但实际相差数像素的伪对齐
```

## 7. 页面填充与留白检查

```text
[ ] 页面是否未仅依赖卡片高度填充画布
[ ] 内容较少时，是否未通过增大卡片 height、min-height 或 padding 撑满页面
[ ] 是否未添加无意义长文、重复信息或纯装饰图形填充空间
[ ] 标题区、主体内容区、辅助视觉区是否分别约占画布高度的 10%～18%、45%～65%、15%～30%
[ ] 连续无功能留白是否不超过画布高度的 25%
[ ] 有效信息与有效视觉元素覆盖面积是否达到画布的 55%～75%
[ ] 留白是否分散在标题、模块和视觉元素之间，未形成大面积空洞
[ ] 低密度页面是否避免“顶部一排自适应卡片 + 下方大面积空白”
[ ] 当卡片下方连续留白接近画布高度的 25% 时，是否已改为流程、路径、关系结构、主视觉数据或主题信息图形
```

## 8. 背景检查

```text
[ ] 是否有低透明青绿色 + 香槟金柔和流线
[ ] 流线是否位于底部或左右侧
[ ] 背景是否不干扰正文
[ ] 是否没有强发光、复杂粒子、HUD 科技线框
```

## 9. 内容密度检查

```text
[ ] 是否一页只讲一个核心结论
[ ] 是否不超过 2~4 个主要模块
[ ] 是否不超过 2~4 个关键数字
[ ] 是否没有大段正文
[ ] 是否没有密集表格
[ ] 远距离是否一眼可读
[ ] 是否未因降低信息密度而删除、改写或合并正式内容
[ ] 是否已在生成前判定为高 / 中 / 低密度内容
[ ] 高密度内容是否使用紧凑卡片、分栏或表格
[ ] 中密度内容是否使用卡片 + 数据图形组合
[ ] 低密度内容是否使用少量大模块、主视觉数据、流程图或主题图形
[ ] 是否未对所有内容密度默认套用三等分卡片布局
[ ] 案例页是否优先体现挑战 → 策略 → 结果的关系叙事
```

## 10. 风格一致性检查

```text
[ ] 是否高级、明亮、克制、商务
[ ] 是否具备轻科技感但不过度炫光
[ ] 是否保留品牌营销 / 生意增长气质
[ ] 是否避免模板感、卡通感、脏乱感
```

## 背景图检查

```text
[ ] 是否按照页面角色使用 cover / content / closing 背景
[ ] 背景图是否适配 3696 × 1008
[ ] 背景图是否只承载氛围，不承载必须编辑的信息
[ ] 主标题、正文、数字、Logo、二维码是否仍为独立可编辑元素
[ ] 内容页背景是否比封面背景更克制，不干扰阅读
```

## 固定品牌信息检查

```text
[ ] 是否没有额外生成第二个“快手联盟”文本
[ ] 是否没有叠加重复 Logo
[ ] 是否没有把顶部品牌区自动生成为常驻文字
[ ] 会议名、年份、演讲人等是否只在原始内容提供时展示
[ ] 背景图自带标识与 HTML 可编辑内容是否没有重复
```

## 独立 Logo 组件检查

```text
[ ] 背景图是否无 Logo
[ ] Logo 是否通过 FixedBrandLogo 独立组件渲染
[ ] 封面 / 内容 / 封尾是否都显示 Logo
[ ] Logo 是否使用 SVG 或高清透明 PNG
[ ] 是否没有重复生成“快手联盟”文字
[ ] 是否没有在 Logo 下方额外生成品牌副标题
[ ] Logo 是否未被背景图压缩导致模糊

## 11. 交付检查

```text
[ ] 每页是否为独立 3696 × 1008 slide，而非纵向长页面
[ ] 100% 画布尺寸下是否无溢出、遮挡、裁切和安全区越界
[ ] 缩放预览下是否仍能辨认标题、关键数字和核心关系
[ ] 标题、正文、数字、图表标签、Logo 与二维码是否均为可编辑 DOM / 独立组件
[ ] 每个正式内容节点是否能追溯到 data-source-id
[ ] 是否已逐页运行 `window.__odLayoutAudit()` 并得到 `pass`
[ ] 是否确认 slide 的 scrollWidth / scrollHeight 未分别超过 3696 / 1008（允许 2px 误差）
[ ] 是否确认每个正式内容节点未越出安全区，且每个文本节点的 scrollWidth / scrollHeight 未超过 clientWidth / clientHeight（允许 2px 误差）
[ ] 是否确认没有通过 `overflow:hidden` 裁切正式文字
[ ] 是否已将 layoutAudit 的页数、节点数与最大溢出值写入 `quality_report.json`
```
