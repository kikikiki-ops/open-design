# Changelog

## 2.6.0 — 2026-07-16

### 新增

- 新增高级关系组件库，覆盖闭环飞轮、能力升级阶梯、同口径前后效果、漏斗转化、决策树、风险/优先级热力图、KPI 塔和可复算瀑布关系。
- 页面计划新增 `advancedRelationSpec`，记录关系类型、来源事实、几何锚点、回退方案与审计项。

### 约束

- 高级图形仅可在来源关系完整时渲染；禁止虚构环线、比例、坐标、分支、分数、增量或算术关系，失败时必须回退为普通关系布局。

## 2.5.0 — 2026-07-16

### 新增

- 新增页面组合与信息链路库，覆盖结论证据、分栏、表格、清单、图表、前后对比、机会矩阵、流程、并行轨道、时间轴、生态桥接、架构、径向能力和多媒体证据。
- 规划页新增 `compositionVariant`、`primaryClaim`、`componentPlan`、`informationLinks` 和 `capacityCheck`，禁止所有内容回退为默认卡片墙。

### 约束

- 连接线必须映射真实因果、证明、比较、阶段、归属或总结关系；多组件同带必须共享父级几何锚点并通过容量检查。

## 2.4.3 — 2026-07-16

### 新增

- 新增“图文搭配与来源配图组件规则”，从市场 PPT 的主图配文本、主图配双证据卡、右侧竖向图片带、双图对照和图片流程提炼为可复用组件。
- 来源图片新增 `must-render` / `optional` / `do-not-render` 分级，承载论据的图片必须进入内容账本、页面计划和独立媒体组件。

### 约束

- 图文模块共享内容带边界；图片保持原始比例并通过媒体审计，禁止拉伸、静默丢弃或使用图片承载不可编辑正文。

## 2.4.2 — 2026-07-16

### 新增

- 新增“先诊断溢出原因”硬规则：安全区越界与内容容量超限必须分流处理。
- 安全区越界强制保持左右 220px、上下 90px；内容容量超限强制按重新分组、更换布局、必要时拆页处理，且正式内容不得删改。

### 修复

- 禁止通过缩小安全区、压缩边距、缩小字号、截断文本或裁切内容来掩盖溢出。

## 2.4.1 — 2026-07-16

### 修复

- 新增“超宽 HTML PPT 溢出与裁切验收协议”，将 `getBoundingClientRect()` 几何检查扩展为每页画布、安全区、正式内容节点与文本节点的强制 DOM 审计。
- 强制交付页暴露 `window.__odLayoutAudit()`；审计未通过或无法执行时，质量报告必须标记为 `blocked`，不得交付为完成状态。
- 明确禁止使用 `overflow:hidden` 裁切正式内容，并定义内容超限时的重排、换模板与拆页修复顺序。
- 将防溢出 CSS 基线和审计接口同步到预览示例。

## 2.4.0 — 2026-07-15

### 修复

- 修复 AutoBoard 仅按新建演示提示执行、无法稳定优化当前 PPTX 的问题。
- 新增 PPTX 输入接收与逐页分类：可编辑、图片型、混合和未知页面。
- 强制优化任务输出 Open Design 可预览的 `index.html` 与 `index.html.artifact.json`，并保留源 PPTX。

## 2.3.2 — 2026-07-15

### 修复

- 将卡片对齐和异构模块对齐规则加入插件 manifest 的显式上下文，确保实际 Skill 注入时加载。
- 在总控与风格绑定契约、页面计划 schema 中加入 `alignmentContract`，传递共同内容带、DOM 几何检查与 2px 误差要求。

## 2.3.1 — 2026-07-15

### 新增

- 新增“同一内容带中的异构模块与卡片对齐”高优先级协议。
- 明确一级模块共同高度 `H`、标题条与卡片区计算、对应行轨道和单体成果/图表卡的等高规则。
- 新增基于 `getBoundingClientRect()` 的 DOM 几何验收，一级模块和对应卡片边界误差上限为 2px。

### 修复

- 防止同带内的前两组列表卡整体偏矮、单张成果卡异常偏高，以及通过局部位移伪造对齐。

## 2.3.0 — 2026-07-15

### 新增

- 基于“快手联盟 — 开发者大会PPT 线稿V2-20260714”新增 20 个可复用页面模板。
- 每个模板定义 `templateId`、页面家族、内容模式、槽位、组件、容量、宽屏/超宽变体、回退模板与禁用场景。
- 新增 `Metrics`、`Charts`、`FlowAndRelationship`、`CaseAndEvidence`、`TaxonomyAndMatrix`、`PageChrome` 组件分类。
- 页面规划新增 `templateId` 和 `templateSelection`，记录槽位覆盖、超宽变体与适配动作。

### 约束

- 明确禁止把 2.67:1 参考稿横向拉伸至 3696 × 1008 / 11:3 交付画布。
- 超宽空间只能用于模块间距、流程轨道、证据/总结侧栏或图表区域；卡片、截图、图片和圆形关系图必须保持比例。

## 2.0.0 — 2026-07-15

### 修复

- 修复章节参考页被当成全局默认页面结构的问题。
- 修复页面角色与页面类型混用导致的封面/章节页误路由。
- 修复 `SectionPage`、`ClosingPage` 在背景规则中出现、但未进入统一页面类型 Schema 的不一致。
- 修复旧版 `GeneralStructuredPage` 兜底规则过弱，仍可能被视觉模板覆盖的问题。
- 修复内容账本缺少坐标、分组、关系目标和渲染策略，导致只提取标题、不理解正文结构的问题。

### 新增

- 页面角色硬门槛与负向排除规则。
- 语义特征分析与路由证据。
- `SectionDividerPage`、`ChartAnalysisPage`、`StructuredContentPage`、`ClosingPage`。
- 结构冻结 `structureFrozen`，风格只能在冻结后绑定。
- 页面内容区域映射 `contentMapping`。
- 关系保真与唯一来源绑定检查。
- 整套 PPT 的 Hero/章节页滥用诊断。
- 规划、渲染、审计四种执行模式。
