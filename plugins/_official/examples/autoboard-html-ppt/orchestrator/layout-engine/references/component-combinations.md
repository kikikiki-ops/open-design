# 组件组合模式

组件不是孤立使用的。以下是高频组合模式，供 AI 根据内容关系直接调用。

---

## 0. 双主卡 + 中轴说明

**适用**：两个对称角色模块，中间有桥梁性概念说明

详细规则见 `references/pattern-dual-cards-center-description.md`。

```text
左主卡 + 中轴说明 + 右主卡
```

**组合**
- A/B 级：左右主卡
- B 级：中轴说明组件
- C 级：标签 / 连接符

---

## 1. 中心构图：中心圆 + 双侧卡

**适用**：平台连接、供需关系、生态协同、中心概念页

```text
header：居中标题
main：
  左侧角色卡（25%）
  中央圆形主容器（50%）
  右侧角色卡（25%）
footer：总结长条
```

**组合**
- A 级：中心圆形承载
- B 级：左右角色卡 / 说明卡
- C 级：标签 / 图标

---

## 2. 双圆公式：圆 + 圆 + 连接符

**适用**：变量乘积、双能力拼接、A×B 公式页

```text
header：居中标题
main：
  左圆（核心变量）
  中间连接符
  右圆（核心变量）
左右两侧或下方补能力卡
footer：总结条
```

**组合**
- A 级：两个圆形主容器
- B 级：左右能力卡 / 结果卡
- C 级：编号标签

---

## 3. 大卡内嵌小组件

**适用**：模型能力、机制说明、复杂模块页

```text
大卡片
├── 标题
├── 内嵌截图 / 图表 / 胶囊标签
├── 说明
└── 结论
```

**组合**
- A/B 级：大圆角卡片
- B 级：截图 / 图表组件
- C 级：胶囊标签

规则：内部小组件依附大卡，不单独跑到页面层。

---

## 4. 左引导 + 右卡片群

**适用**：未来展望、方向拓展、应用场景扩展

```text
main：
  左侧引导文案（17%–25%，无底框）
  右侧 3–5 张业务卡（75%–83%）
```

**组合**
- B 级：左侧无底框主题区
- A/B 级：右侧业务卡片
- C 级：卡内图标

---

## 5. 中央图片 + 双侧数据

**适用**：案例展示、UI 截图、产品素材页

```text
main：
  左数据（17%–20%）
  中央图片组（60%–66%）
  右数据（17%–20%）
```

**组合**
- A 级：图片主体区（无重卡）
- B 级：左右数据说明区
- C 级：顶部分类标签

规则：图片自己就是主体，不强上底框。

---

## 6. 路径 + 节点 + 旁注

**适用**：战略路线图、增长曲线、能力演进

```text
main：
  路径线为主骨架
  节点圆 / 节点卡沿路径分布
  说明文字放在节点外侧
```

**组合**
- A 级：路径节点圆
- B 级：输入卡 / 结果卡
- C 级：胶囊标签 / 小注释

规则：节点承载核心词，解释文字尽量放外侧。

---

## 7. 矩阵列 + 底部胶囊结论

**适用**：能力矩阵、分类总览、组织能力地图

```text
main：
  3–5 列等宽栏目
  栏目内部：标题 → 条目 → 底部胶囊总结
```

**组合**
- B 级：无底框栏目结构
- C 级：图标 + 条目 + 胶囊总结

规则：栏目间靠对齐和分隔线，不靠大卡片。

---

## 8. 对称结果 + 异形描边框

**适用**：结果页、过渡页、章节重点结论

```text
main：
  左结果框（镜像）
  中央核心结论
  右结果框（镜像）
```

**组合**
- A 级：左右异形描边框
- B 级：中央无底框结论

规则：适合信息量少但需要强张力的页。

---

## 9. 同心圆 + 侧数据卡

**适用**：用户价值分层、能力沉淀、外围到核心

```text
main：
  左侧主题引导
  中央同心圆模型
  右侧数据卡列
```

**组合**
- A 级：同心圆主容器
- B 级：右侧数据卡
- B 级：左侧无底框主题文案

---

## 9.5 三段流程 + 连接节点

**适用**：三阶段流程、三段能力推进、每段之间需要明确方向连接

详细规则见 `references/pattern-triple-process-with-connectors.md`。

```text
模块 A + 连接节点 + 模块 B + 连接节点 + 模块 C
```

**组合**
- A/B 级：三张流程模块卡
- B 级：中间模块可加权放大
- B/C 级：两个连接节点组件（圆底 + 大箭头）

---

## 10. 三卡数据 + 底部总结

**适用**：成绩页、增长页、三项结果总览

```text
main：
  三张等宽数据卡
footer：统一总结
```

**组合**
- A/B 级：三张数据卡
- B 级：底部总结长条或居中总结

规则：三卡应尽量同结构、同宽度、同层级。
