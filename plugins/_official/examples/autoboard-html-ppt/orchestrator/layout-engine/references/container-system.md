# 容器承载系统

容器不是装饰，而是信息结构的一部分。排版前先判断内容要落在哪种容器里。

---

## 1. 容器等级

| 等级 | 名称 | 用途 | 信息权重 |
|------|------|------|----------|
| A | 主容器 | 承载页面核心概念 / 核心模块 / 主视觉 | core |
| B | 次容器 | 承载说明模块 / 辅助结果 / 过渡信息 | normal |
| C | 辅助容器 | 承载标签 / 状态 / 编号 / 小结论 | aux |

判断原则：
- 一个页面最多 1–2 个 A 级主容器
- B 级容器可重复出现，但不应抢过 A 级
- C 级容器只做索引和补充，不承载正文

---

## 2. 容器类型

### 2.1 圆形承载（A）

**适用**：中心概念、公式变量、路径节点、关键状态点

**内部结构**

```text
圆内：
主文案（L3）
次文案（L4，可有可无）

圆外：
补充解释（L4）
```

**规则**
- 圆内不要放长段文字
- 适合承载“词”或“短句”
- 若说明较长，放到圆外侧

### 2.2 同心圆承载（A）

**适用**：分层关系、用户沉淀、价值分层、能力递进

**规则**
- 中心承载最高优先级概念
- 外环承载次级概念
- 文字沿环分布，不额外加卡片

### 2.3 大圆角卡片（A/B）

**适用**：完整模块信息、数据+说明+图表、模型能力块、结果块

**内部结构**

```text
标签/标题
→ 核心内容（数字/图片/图表）
→ 说明
→ 结论
```

**规则**
- 一个卡片只表达一个主要主题
- 适合承载完整信息组
- A 级卡用于主模块，B 级卡用于次模块

### 2.4 轻量说明卡（B）

**适用**：阶段说明、背景信息、过渡说明、次级结论

**规则**
- 视觉明显弱于主卡
- 底色更浅、边界更轻
- 文本量可稍多，但不抢主视觉

### 2.5 胶囊承载（C）

**适用**：标签、状态、编号、列级小结论、角色标识

**规则**
- 短文本单行优先
- 不承载复杂正文
- 常贴附在主容器边缘或顶部

### 2.6 长条承载（A/B）

**适用**：页面总结、下一步方向、横向关系收束、底部总览

**内部结构**

```text
图标/标签
→ 结论主句
→ 下一步 / 扩展说明
```

**规则**
- 通常横向展开，放 footer 或 main 底部
- 内容以一句完整判断为主

### 2.7 异形描边承载（A/B）

**适用**：结果强调、方向扩展、战略重点、对称结论页

**规则**
- 信息量不要太大
- 适合“1 个数字 + 1 个标题 + 1 段说明”
- 负责制造张力和方向感

### 2.8 无底框结构承载（B/C）

**适用**：目录、左主题引导、矩阵栏目、中轴式排版

**规则**
- 依靠对齐、分隔线、留白成立
- 不是所有页面都要上卡片
- 当页面已有强主视觉时，辅助区优先无底框

---

## 3. 容器与层级映射

| 层级 | 推荐容器 |
|------|----------|
| L0 | 无底框标题区 / 中轴标题组 |
| L1 | 标题下方副结论 / 长条承载 |
| L2 | 卡片标题 / 栏目标题 / 角色标题 |
| L3 | 圆形主容器 / 数据卡主数字 / 同心圆中心 |
| L4 | 说明卡 / 卡片正文 / 圆外解释 |
| L5 | 胶囊标签 / 小标签 / 角落注释 |

---

## 4. 使用决策树

```text
如果内容是页面唯一核心概念 → 圆形 / 同心圆
如果内容是一组完整模块 → 大圆角卡片
如果内容是阶段说明 / 背景说明 → 轻量说明卡
如果内容是标签 / 编号 / 状态 → 胶囊
如果内容是一句话总结 → 长条承载
如果页面强调张力或方向扩展 → 异形描边
如果只是辅助引导 / 留白构图 → 无底框结构
```
