# 图片与内容组合规范

图片不是孤立元素，它必须和文字、数据、图表形成明确的组合关系。以下是所有涉及图片的布局策略。

---

## 1. 图片的角色判定

拿到需求中有图片时，先判定图片在页面中承担的角色：

| 图片角色 | 特征 | 空间策略 |
|----------|------|----------|
| **证据** | 证明某个数据/结论的真实性 | 图片跟随数据，不独立占大区域 |
| **主体** | 图片本身就是页面核心信息（如产品界面、案例截图） | 图片占 main 55%–75%，文字围绕 |
| **说明** | 解释模型/机制/流程的工作方式 | 图片嵌入说明模块内部，和文字并列 |
| **氛围** | 背景/品牌/装饰性 | 不参与布局计算，叠加在底层 |

判定规则：
- 图片内有产品界面、截图、数据面板 → 主体
- 图片旁有数据结果且数据是核心 → 证据
- 图片在流程节点或模型描述内部 → 说明
- 不属于以上三种 → 氛围（降级处理）

---

## 2. 图片占位比例

图片在 main 区域中的宽度占比，由图片角色和数量共同决定：

| 角色 | 1 张 | 2–3 张 | 4–6 张 | 7+ |
|------|------|--------|--------|-----|
| 主体 | 60%–75% | 55%–70% | 50%–65% | 分组/拆页 |
| 证据 | 25%–40% | 30%–50% | 40%–55% | 分组 |
| 说明 | 嵌入模块内部，不独立占列 | 同左 | 同左 | — |

**图片高度**：占 main 区域的 50%–70%（主体图片可到 75%）

## 3. 图文组合布局

### 3.1 图左 + 数据右（证据型）

图片作为证据，右侧放数据结果：

```text
main (100%)
├── 图片区 (40%)        ← 证据图片
│   ├── 分类标签 (上)
│   └── 图片组 (主体)
└── 数据区 (60%)        ← 核心数据 + 结论
    ├── 核心数字
    ├── 指标说明
    └── 结论
```

适用：案例展示 + 增长数据（如 AIGC 素材 + CTR 增长）

### 3.2 数据左 + 图右（证据型，镜像）

数据是核心，图片佐证：

```text
main (100%)
├── 数据区 (55%)        ← 核心数据（L3 级）
│   ├── 标题
│   ├── 核心数字
│   └── 说明
└── 图片区 (45%)        ← 证据图片
```

适用：增长指标 + 截图证据

### 3.3 图中央 + 数据双侧（主体型）

图片是页面核心，两侧放数据：

```text
main (100%)
├── 左数据 (17%–20%)   ← 一个关键指标
├── 图片组 (60%–66%)   ← 多张案例图横向排列
└── 右数据 (17%–20%)   ← 另一个关键指标
```

适用：多个产品界面 + 两侧关键数据

### 3.4 图上方 + 文字下方（说明型）

图片解释机制/模型，下方配文字说明：

```text
main 列内子布局
├── 图片区 (55% 高)    ← 模型图/流程图/机制图
└── 文字区 (45% 高)    ← 模块标题 + 说明 + 结论
```

适用：模型能力页、机制说明页

### 3.5 图嵌入卡片（说明型）

图片嵌入内容卡片内部，图文一体：

```text
卡片内部结构
├── 卡片标题
├── 图片/截图 (占卡片 40%–60% 高)
├── 说明文字
└── 结论
```

适用：SDK 能力升级、产品功能展示

### 3.6 纯图片序列（主体型，无侧数据）

图片本身就是完整信息：

```text
main (100%)
├── 标签/分类 (上方)
├── 图片横向序列 (55%–75% 高)
└── 底部结论条
```

适用：纯案例展示、素材总览

## 4. 多图排列规则

### 横向序列

```text
图片1 | 图片2 | 图片3 | 图片4
  ↑       ↑       ↑       ↑
 等宽    等宽    等宽    等宽
 统一间距（占图片区宽度 1%–2%）
```

- 统一宽高比
- 统一间距
- 图片之间不留大段文字
- 超过 6 张时分组

### 左右对比

```text
图片A (48%)  |  图片B (48%)
   ↑               ↑
 过去/旧方案     现在/新方案
```

- 两张图片必须同比例
- 各配标题和关键说明
- 底部统一结论

### 网格

```text
图片1 | 图片2
图片3 | 图片4
```

- 2×2 为主，不超过 3×3
- 统一尺寸
- 每张配短标签

---

## 5. 图文间距规范

| 位置 | 间距 |
|------|------|
| 图片上方与标签 | 1%–2%（main 高度） |
| 图片下方与说明文字 | 2%–3% |
| 图片左右与数据区 | column_gap（1.5%） |
| 图片之间（横向序列） | 1%–2%（图片区宽度） |
| 图片内部与外部文字 | 图片不自带文字标注时，外部说明距图 2% |

---

## 6. 图片在组件中的组合方式

| 组件 | 图片如何嵌入 |
|------|-------------|
| C02 核心数据 | 图片不嵌入，放在相邻列 |
| C04 内容卡片 | 图片占卡片内 40%–60% 高度，位于标题下方 |
| C05 图表 | 图表本身就是"图"，不额外嵌图片 |
| C06 图片/案例 | 独立使用，图片为主体 |
| C07 流程节点 | 图片作为节点内的小型证据图（占节点高 30%–40%） |
| C09 结论条 | 不嵌入图片 |

---

## 7. 图片自适应规则

| 情况 | 处理 |
|------|------|
| 图片少（1–2 张）| 放大单图，配充足说明 |
| 图片多（4–6 张）| 缩小单图，统一排列，减少外部文字 |
| 图片内部有文字 | 不重复写外部说明，用标签标注即可 |
| 图片比例不一致 | 统一裁切为同一比例，不拉伸 |
| 图片质量低 | 缩小展示区域，加大文字占比 |
| 无图片但有截图需求 | 文字为主，标注"待补图"位置 |
