# 内容拆解详细规范

## 拆解维度

对每条内容标注：

| 维度 | 含义 | 可选值 |
|------|------|--------|
| type | 内容类型 | title / data / description / stage / comparison / case / chart / conclusion |
| weight | 信息权重 | core（必须在页面突出）/ normal（常规展示）/ aux（辅助补充） |
| format | 表现形式 | text / number / percentage / image / chart / icon |
| relation | 与其他内容的关系 | parallel / compare / cause / aggregate / sequence |

## 拆解流程

```text
1. 找出所有标题性内容 → type=title
2. 找出所有数字/百分比 → type=data, format=number/percentage
3. 找出所有说明性段落 → type=description, format=text
4. 找出所有阶段/步骤 → type=stage
5. 找出所有对比性内容 → type=comparison
6. 找出所有案例/图片 → type=case, format=image
7. 找出所有结论/判断 → type=conclusion
8. 判断每条内容的 weight
```

## weight 判断规则

- 核心数字（增长率、关键占比、突破性成果）→ core
- 核心结论（整页最想传达的判断）→ core
- 模块标题、场景说明 → normal
- 注释、来源、辅助标签 → aux
- **core 项不超过 3 个**，超过时降级为 normal

## 层级分配流程

```text
1. core 中最重要的 1 个 → L0（页面标题）
2. core 中 1–2 个结论性内容 → L1
3. normal 模块的标题 → L2
4. core 数字/百分比 → L3
5. normal 说明文字 → L4
6. aux 内容 → L5
```

## 层级定义

| 层级 | 角色 | 空间分配倾向 |
|------|------|--------------|
| L0 | 页面主标题（一个） | 占 header 全部，最大视觉权重 |
| L1 | 页面核心结论（1–2 个） | header 内主标题旁，仅次于 L0 |
| L2 | 一级模块标题 | 各模块顶部，区分功能区域 |
| L3 | 核心数据/核心概念 | 模块内视觉焦点，数字比说明大 3–5 倍 |
| L4 | 正文说明/图表说明 | 常规文字，辅助理解 |
| L5 | 注释/来源/辅助标签 | 最小最淡，角落或底部 |

## 层级驱动空间

| 层级组合 | 空间行为 |
|----------|----------|
| L3 数据突出 | 该模块宽度放大（如 1.5:1 非等分） |
| L4 文字多 | 该模块高度增加 |
| L5 为主 | 整个模块缩减为小标签条 |
| 无 L3 数据 | 不留图表区域，纯文字排版 |

## 混合关系处理

内容常有混合关系。处理方法：

| 混合类型 | 处理方式 |
|----------|----------|
| 并列 + 因果 | 主模板用并列（T02/T08），因果用箭头或路径线串联 |
| 对比 + 数据 | 主模板用对比（T04），数据作为两侧结果卡 |
| 递进 + 聚合 | 主模板用递进（T06），中心节点作聚合锚点 |
| 分层 + 案例 | 主模板用分层（T06 同心圆），案例作外围证据 |
| 数据 + 展望 | 主模板用数据（T02），展望结论放 footer |
| 对比 + 流程 | 主模板用对比（T04），流程线连接两侧 |

判断原则：
1. 找出占比最大的关系 → 主关系，决定模板
2. 次要关系 → 在主模板内局部调整
3. 零散内容 → 降级为 footer 或辅助信息
