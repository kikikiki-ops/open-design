# 异构模块对齐规则

当页面同时包含列表、卡片、箭头、数据指标、图表或总结模块等不同类型模块时，不得简单要求全部元素等宽或等高。必须先建立明确的对齐锚点，再进行内部排版。

## 1. 外边界锚点

- 同一逻辑层级模块共享顶部或左侧基线。
- 总结型模块至少与主体模块共享一个外边界。
- 当多个一级模块属于同一横向内容带时，必须进一步共享顶部、底部和整体高度；此时共同内容带高度 `H` 优先于本规则中“不同模块可不同高”的一般权限。详见 `card_alignment.md` 的“同一内容带中的异构模块与卡片对齐”。

## 2. 行轨道锚点

- 有逐项对应关系的两个列表，必须共享相同行数、行高和行间距。
- 对应项的顶部、底部和中心线必须一致。

## 3. 中心线锚点

- 箭头、连接线和转折符号必须对准连接对象中心，而不是页面中心。
- 核心数字、图表或结论内容块，应与主体信息区域的视觉中心对齐。

## 4. 语义关系优先

- 有对应关系的元素优先逐行对齐。
- 有因果关系的元素优先通过连接线和中心线对齐。
- 有总结关系的元素优先通过共同顶部和内容中心对齐。

## 5. 不同类型模块

- 不同模块可不同宽、不同高，但必须至少共享一个边界锚点和一个内容锚点。
- 不允许完全独立摆放。
- 例外：位于同一横向内容带的一组一级模块，必须等高且共享顶部、底部边界；不同的是内部结构，而不是外部框体。

## 6. 父级控制与禁止项

- 应回到父级 Grid、Flex、行轨道和对齐变量中解决对齐问题。
- 禁止通过单独设置 `margin`、`top`、`left` 或 `translate` 补偿式定位。

## 7. 生成后检查

- 对应项逐行齐平。
- 箭头真正对准连接对象。
- 总结模块与主体模块建立共同基线。
- 不存在视觉接近、实际相差数像素的伪对齐。
- 不同模块形成明确的视觉阅读路径。
- 同一内容带内的一级模块 `top`、`bottom`、`height` 差值均不超过 2px；复合列表和对应行卡片的几何数据也已通过 DOM 检查。
