# Test Plan V2

## 1. 推荐顺序

1. `plan` 模式跑整套 PPT，只验证内容和页面路由；
2. 人工抽查高风险页；
3. 组件单测；
4. 风格单测；
5. 代表页集成；
6. 整套 `full` 模式回归。

## 2. 高风险页

必须覆盖：

- 首屏但包含业务数据的页面；
- 带大号数字但不是章节编号的页面；
- 标题含“增长、升级、策略、预算”的内容页；
- 多指标、多模块和流程页；
- 图表或图片主导页；
- 章节页和封尾页。

## 3. 验收指标

- 页面角色准确率；
- 页面类型准确率；
- Hero 误用率 = 0；
- 正式内容覆盖率 = 100%；
- 数字保真率 = 100%；
- 编辑备注误渲染率 = 0；
- 溢出率 = 0。

## 4. 失败案例回归

“05 深度预算引擎升级”页面若同时存在多组预算能力和指标：

- 预期 `pageRole = content`；
- 预期内容型页面与内容型布局；
- 禁止 `SectionDividerPage + HeroLayout`。
