# Quality Check V2

## 1. Planner Gate

渲染前必须检查：

- 所有源页都已建立页面分析；
- 优化 PPT 时，已生成 `intake_result.json`，每张源页已分类为 editable / image-based / mixed / unknown，且 blocked 页面未被静默跳过；
- 所有正式内容进入内容账本；
- 页面角色硬门槛通过；
- Hero 类型未被当作兜底；
- 页面类型有明确证据；
- 每个 `must-render` 项已进入 `contentMapping`；
- 布局容量未超限；
- 每页已完成高 / 中 / 低内容密度判定，且布局与密度匹配；
- 每页已选择与 `pageType` 匹配的 `templateId`，所有 `requiredSlots` 均有内容映射；
- 模板的 `contentCapacity`、`allowedComponents`、`fallbackTemplates` 和 `prohibitedUses` 已检查；
- 交付页声明 `canvasVariant = ultrawideVariant`，并记录非拉伸适配动作；
- `structureFrozen = true`。

Planner Gate 失败时不得渲染。

## 2. Renderer Gate

- 所有 `must-render` 内容在 HTML 中出现；
- 优化 PPT 时，项目根目录存在可预览的 `index.html` 与同级 `index.html.artifact.json`，源 `.pptx` 未被覆盖；
- 数值、单位、百分号、箭头和条件完全一致；
- 没有未经授权的改写或新增；
- 分组、流程、因果和对比关系保持；
- 编辑备注未误渲染；
- 无溢出、遮挡和安全区越界；
- 字体达到大屏最小可读要求；
- HTML 可编辑；
- 风格未修改冻结结构；
- Logo 为独立清晰组件。
- 每页是独立 3696 × 1008 slide，而非纵向长页面；
- 在 100% 画布与缩放预览下均无溢出、遮挡、裁切和安全区越界；
- 每个溢出问题已先记录为“安全区越界”或“内容容量超限”：前者保持左右 220px、上下 90px 安全区并修正共同父级边界；后者已通过重新分组、更换布局或拆页解决，未压缩安全区、字号或正式内容；
- 会议名称、日期、演讲人、联合品牌和二维码没有被无依据补充。
- 页面未通过拉伸卡片或添加无意义内容填充画布；连续无功能留白不超过画布高度的 25%。
- 每个内容页已声明 `primaryClaim`、`compositionVariant`、`componentPlan`、`informationLinks` 和 `capacityCheck`；所选组合符合页面关系，未将流程、对比、表格、媒体证据或层级关系退化为默认卡片墙。
- 使用闭环、升级阶梯、前后效果、漏斗、决策树、热力图、KPI 塔或瀑布图的页面已声明 `advancedRelationSpec`，并通过来源事实、算术关系和共同轨道审计；未虚构闭环、比例、坐标、条件分支、分数或增量。
- 标题、主体内容与辅助视觉的占比符合风格检查，且有效信息与有效视觉元素覆盖面积达到画布的 55%～75%。
- 低密度页面不存在“顶部一排自适应卡片 + 下方大面积空白”；如卡片下方连续留白接近画布高度 25%，已改为流程、路径、关系结构、主视觉数据或主题信息图形。
- 没有将 2.67:1 的参考构图直接横向拉伸到 3.67:1；图片、截图、卡片、图表符号和圆形关系图均保持比例，额外宽度由轨道、模块间距、侧栏或图表域吸收。
- 同一横向内容带的一级模块已由共同父级内容带高度 `H` 控制，通常为画布高度的 35%～55%；一级模块 top、bottom、height 的 DOM 几何误差均不超过 2px。
- 复合“标题条 + 卡片列表”模块使用内部 Grid 将卡片区等分；逐项对应卡片共享行轨道。单体数据、成果和图表模块与同带模块等高，未通过空容器扩大制造强调。
- 对齐失败只通过共同父级 `H`、`align-items: stretch`、一级模块尺寸规则或内部 Grid 行轨道修复；不存在局部 margin、定位、translate、独立 height/min-height 或无意义 padding 补偿。
- 已对每页运行 `window.__odLayoutAudit()`；审计结果为 `pass`，且结果已写入 `quality_report.json`。无法运行审计时必须为 `blocked`，不得以截图或肉眼检查替代。
- 审计已确认每页 `scrollWidth <= 3696 + 2`、`scrollHeight <= 1008 + 2`；正式内容未越出安全区；所有文本节点 `scrollWidth <= clientWidth + 2` 且 `scrollHeight <= clientHeight + 2`。
- 承载正式内容的卡片、列表、指标、流程节点未使用 `overflow:hidden` 裁切文字。
- 受限内容带内的纵向卡片组已使用显式 `repeat(N, minmax(0, 1fr))` 行轨道，最后一张卡底边未超出父级内容带 2px；未让默认 `auto` 行轨道推动页面溢出。

## 3. 反封面诊断

以下任意情况默认失败：

- 非首屏被判为 `CoverPage` 且无用户指令；
- 内容页使用 `HeroLayout`；
- Hero 页含正式指标、图表、流程或多组业务模块；
- 连续两个及以上 Hero 页面；
- 多个语义不同的页面都被压缩为“大编号 + 居中标题 + 一句副标题”；
- `StructuredContentPage` 被渲染为章节版式。
- 横向语义卡片组未顶边、底边对齐或外框未等高；纵向组未等宽；矩阵组未统一行高、列宽或间距。
- 同组卡片逐张设置位置、位移、外边距、box-sizing 或尺寸计算方式，导致实际边界误差大于 2px。
- 异构模块未建立共同外边界、行轨道或中心线锚点；对应列表未逐行齐平，或箭头、连接线未对准连接对象中心。
- 总结模块未与主体模块建立共同外边界，或以单独 `margin`、`top`、`left`、`translate` 制造伪对齐。
- `templateId` 的必填槽位为空、超过内容容量、使用未允许组件，或命中 `prohibitedUses`。
- 模板以“参考第 N 页”的一次性命名或实现出现，未抽象为页面布局、内容模式和可复用组件。
- 同一内容带中的一级模块边界误差、同组卡片高度误差或对应行卡片边界误差超过 2px，或未使用 `getBoundingClientRect()` 完成几何验收。
- 任一 slide、正式内容节点或文本节点发生画布越界、安全区越界、滚动尺寸溢出、裁切或被隐藏内容；或 `quality_report.json` 缺失 `layoutAudit.status = pass`。
- 高级关系图缺少其必要来源事实，或其环线、箭头、节点、坐标、比例、数值、阶段顺序与 `advancedRelationSpec` 不一致。
- 文本卡片未设置 `max-width`，因画布过宽导致正文行长超过约 40 个汉字（34ch）。
- 图片容器未使用 `object-fit: cover`，出现上下或左右无意义空白、图片悬浮在容器中、或拉伸变形。
- 同组图片高度不一致、显示比例不统一、裁切方式不同。
- 单个内容模块宽度超过预设最大宽度，或主体内容从左安全区连续铺到右安全区。

## 4. 内容覆盖

要求：

```text
must-render 覆盖率 = 100%
关键数字覆盖率 = 100%
未经授权改写 = 0
虚构内容 = 0
关系破坏 = 0
```

## 5. 失败恢复

按顺序：重组 → 换变体 → 换内容型页面类型 → 拆页 → 待确认。

禁止：删内容、缩到不可读、改成 Hero 页。
