# Quality Check V2

## 1. Planner Gate

渲染前必须检查：

- 所有源页都已建立页面分析；
- 所有正式内容进入内容账本；
- 页面角色硬门槛通过；
- Hero 类型未被当作兜底；
- 页面类型有明确证据；
- 每个 `must-render` 项已进入 `contentMapping`；
- 布局容量未超限；
- `structureFrozen = true`。

Planner Gate 失败时不得渲染。

## 2. Renderer Gate

- 所有 `must-render` 内容在 HTML 中出现；
- 数值、单位、百分号、箭头和条件完全一致；
- 没有未经授权的改写或新增；
- 分组、流程、因果和对比关系保持；
- 编辑备注未误渲染；
- 无溢出、遮挡和安全区越界；
- 字体达到大屏最小可读要求；
- HTML 可编辑；
- 风格未修改冻结结构；
- Logo 为独立清晰组件。

## 3. 反封面诊断

以下任意情况默认失败：

- 非首屏被判为 `CoverPage` 且无用户指令；
- 内容页使用 `HeroLayout`；
- Hero 页含正式指标、图表、流程或多组业务模块；
- 连续两个及以上 Hero 页面；
- 多个语义不同的页面都被压缩为“大编号 + 居中标题 + 一句副标题”；
- `StructuredContentPage` 被渲染为章节版式。

## 4. 内容覆盖

要求：

```text
must-render 覆盖率 = 100%
关键数字覆盖率 = 100%
未经授权改写 = 0
虚构内容 = 0
关系破坏 = 0
```

## 5. 失败恢复

按顺序：重组 → 换变体 → 换内容型页面类型 → 拆页 → 待确认。

禁止：删内容、缩到不可读、改成 Hero 页。
