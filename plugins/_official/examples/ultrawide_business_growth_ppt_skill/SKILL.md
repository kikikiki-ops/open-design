# 超宽大屏商务增长风格 PPT Skill

## 适用场景

当用户需要基于「超宽会场大屏 / 品牌营销大会 / 商业化业务汇报 / 数据增长发布 / 案例复盘」生成或改写 PPT 页面时，使用本 Skill。

本 Skill 固定适配以下画布：

- 物理尺寸：11m × 3m
- 像素分辨率：3696 × 1008 px
- 画面比例：11:3（约 3.67:1）
- 输出方向：超宽横屏

视觉基准来自用户提供的快手品牌营销大屏样稿，整体方向为：浅暖底、柔光流线、香槟金商务感、青绿色轻科技感、深青黑标题、白色圆角卡片、低信息密度、中心聚焦式表达。

---

## 设计目标

生成页面时，不要把普通 16:9 PPT 横向拉伸，而要按照「会场超宽主屏」设计。

核心目标：

1. 远距离一眼可读。
2. 一页只讲一个核心结论。
3. 中间承载主信息，左右用于氛围延展。
4. 通过白色圆角卡片承载结构化内容。
5. 用香槟金表达商业成果，用青绿色表达智能策略 / 数据增长。
6. 画面保持高级、明亮、克制、商务、轻科技。

---

## 工作流程

### Step 1：识别页面意图

先判断用户输入内容更适合哪类页面：

- 封面页
- 章节过渡页
- 案例成果页
- 数据结论页
- 策略分析页
- 三段式叙事页
- 指标成果页
- 问题-解法页
- 资源组合页
- 总结收口页

如果内容很多，优先提炼核心结论，不要把所有内容塞进一页。

---

### Step 2：确定页面结构

根据页面类型选择对应结构：

#### 封面页

结构：

- 顶部品牌区
- 中心大标题
- 可选副标题
- 底部柔光流线

适合：会议主题、品牌主题、发布会主视觉。

#### 章节过渡页

结构：

- 顶部品牌区
- 中心章节标题
- 极少量辅助说明
- 底部轻氛围背景

适合：章节切换、议题转场。

#### 案例成果页

结构：

- 顶部品牌区
- 居中主标题
- 中部三段式内容：案例背景 / 方法策略 / 成果结果
- 右侧或中部突出关键数字

适合：品牌案例、项目复盘、业务成果。

#### 数据结论页

结构：

- 居中主标题
- 主图表或核心逻辑图
- 2~4 个关键数字
- 少量辅助说明

适合：趋势洞察、相关性分析、增长结果、人群价值说明。

#### 策略分析页

结构：

- 居中主标题
- 中心问题 / 目标
- 左右展开「问题 - 解法 - 结果」
- 可使用标签、箭头、卡片映射

适合：策略升级、投放解法、资源组合、方法论表达。

---

### Step 3：应用固定画布与安全区

必须使用：

```text
画布：3696 × 1008 px
左安全边距：220 px
右安全边距：220 px
上安全边距：90 px
下安全边距：90 px
核心内容区：3256 × 828 px
```

核心信息必须放在安全区内。左右边缘主要做背景氛围，不放重要文字。

---

### Step 4：应用视觉风格

必须使用以下视觉原则：

- 背景为浅暖白 / 米白。
- 底部或左右两侧有低透明度青绿色 + 香槟金柔光流线。
- 主标题居中，使用深青黑。
- 主体内容使用白色大圆角卡片。
- 卡片边框极细，阴影极轻。
- 关键数字用香槟金或深青黑放大展示。
- 图表用青绿色为主，香槟金为辅。
- 信息密度低，远距离可读。

详细 token 见 `style.md`。

---

### Step 5：控制内容密度

单页建议上限：

- 1 个主标题
- 1 个核心结论
- 2~4 个主要模块
- 1 张主要图表
- 2~4 个大数字
- 5 个以内辅助标签
- 30~80 字正文说明

内容过多时，处理顺序：

1. 提炼主结论
2. 合并同类项
3. 删除重复解释
4. 拆成多页
5. 弱化辅助内容

严禁：缩小字号硬塞、压缩边距、密集表格、满屏小字。

## 输出要求

### 如果输出 HTML 型 PPT 页面

必须：

- `.slide` 容器使用固定尺寸：`width: 3696px; height: 1008px;`
- `html, body, .slideshow` 使用 `width:100vw; height:100vh; overflow:hidden;`
- **必须加入等比自适应缩放脚本**（见下方模板），确保任意屏幕尺寸下 3696×1008 都能完整等比居中显示
- 使用 `position:absolute` 绝对布局，`transform-origin: 0 0`
- 所有核心内容位于安全区内（左右各 220px，上下各 90px）
- 背景流线为装饰层，不影响内容层
- 字号必须满足大屏阅读，主标题建议 60~72px
- 不要出现页面滚动
- 不要出现内容溢出

#### 必须包含的自适应缩放脚本模板

**每一个生成的 HTML PPT 都必须在 `</body>` 前包含以下脚本**（可根据是否有导航栏调整 `NAV_H`）：

```html
<script>
(function () {
  var SLIDE_W = 3696, SLIDE_H = 1008;
  var slides = Array.from(document.querySelectorAll('.slide'));

  function scaleSlides() {
    var navEl = document.getElementById('nav-hud');
    var navH  = navEl ? navEl.offsetHeight : 0;
    var vw    = window.innerWidth;
    var vh    = window.innerHeight - navH;
    var scale = Math.min(vw / SLIDE_W, vh / SLIDE_H);
    var ox    = (vw - SLIDE_W * scale) / 2;
    var oy    = (vh - SLIDE_H * scale) / 2;
    slides.forEach(function (s) {
      s.style.transform       = 'scale(' + scale + ')';
      s.style.transformOrigin = '0 0';
      s.style.left            = ox + 'px';
      s.style.top             = oy + 'px';
    });
  }

  document.addEventListener('DOMContentLoaded', scaleSlides);
  window.addEventListener('load',   scaleSlides);
  window.addEventListener('resize', scaleSlides);
})();
</script>
```

**不允许**使用 `width:100vw; height:100vh` 直接拉伸 `.slide` 容器，这会破坏 3.67:1 超宽比例。

## 禁止项

生成时避免：

- 赛博蓝紫
- 高饱和霓虹光
- 强发光边缘
- 复杂 HUD 科技线框
- 大面积深色底
- 过厚投影
- 过多渐变块
- 拟物按钮
- 花哨字体
- 卡通插画感
- 杂乱粒子
- 大段长文
- 小字号密集表格
- 平均铺满整条长屏的碎信息

---

## 质量检查

生成后必须检查：

```text
[ ] 画布是否为 3696 × 1008
[ ] 是否按照超宽大屏构图，而不是 16:9 拉伸
[ ] 主标题是否居中且醒目
[ ] 核心内容是否集中在中央安全区
[ ] 左右两侧是否主要承担氛围延展
[ ] 背景是否为浅暖白 / 米白
[ ] 是否有柔和青绿色 + 香槟金流线
[ ] 主体是否使用白色大圆角卡片
[ ] 卡片边框和阴影是否克制
[ ] 是否突出关键数字和核心结论
[ ] 图表是否简洁清晰
[ ] 页面信息是否低密度
[ ] 是否避免赛博、炫光、复杂装饰和模板感
[ ] 是否满足远距离观看可读性
```

## 一句话风格提示词

采用超宽大屏商务增长风格，画布为 3696×1008，适配 11m×3m 会场主屏。背景为浅暖白和米白色，底部与两侧加入低透明度青绿色与香槟金柔和流线。主标题居中，信息重心集中在中间区域，左右作为氛围延展。主体内容使用白色大圆角卡片承载，边框细、阴影极轻。颜色以深青黑文字、香槟金重点和青绿色点缀为主。突出大数字、关键结论与简洁图表，信息密度低，远距离易读。避免赛博炫光、复杂科技 HUD、厚重阴影、高饱和渐变、密集小字和拥挤排版。

## 固定背景图系统

本风格 Skill 支持固定封面、内容页、封尾三类背景图。背景图资产由本 Skill 管理，详见 `background_system.md` 与 `asset_manifest.json`。

页面背景枚举：

```text
cover：封面页背景
content：正文内容页背景
closing：封尾页背景
```

分工规则：

```text
风格 Skill：定义背景图资产、视觉强度和使用规则
总控 Skill：判断页面角色并传递 backgroundVariant
组件 Skill：根据 backgroundVariant 渲染背景图
```

背景图不得承载必须可编辑的信息。主标题、正文、关键数字、单位、图表标签、Logo、二维码等应作为独立 HTML / 组件呈现。

## 固定品牌信息禁用规则

当前背景图资产本身已经包含左上角品牌标识，因此本风格 Skill 不应额外生成任何常驻品牌文字、常驻 Logo 组件或顶部小标签。

默认禁止在页面中自动叠加：

- 左上角第二个“快手联盟”
- 顶部常驻品牌文字
- 顶部常驻业务线文字
- 固定角标
- 未来自原始 PPT 正式内容的品牌副标识

如果原始 PPT 页面本身明确提供了品牌文字、会议名称、年份、演讲人信息等，才允许作为可编辑 HTML 文本输出。否则不得凭风格 Skill 自动补充。

## 背景无 Logo 与独立 Logo 组件规则

当前版本采用“背景图不带 Logo，Logo 独立组件渲染”的方案。

必须遵守：

- 背景图只承载流线、光影、色块、纹理与空间氛围
- Logo 不得烘焙进背景图
- Logo 必须通过独立组件渲染
- 封面 / 内容 / 封尾均显示 Logo
- Logo 资产优先使用 SVG
- 不得在 Logo 下方额外生成第二行“快手联盟”文字

Logo SVG 源地址：

```text
https://p5-ad.adkwai.com/udata/pkg/ks-ad-fe/md-tools/quick-cut/6689:5117.b90a9c9d53932e9b.svg
```

默认组件能力：`FixedBrandLogo`。
