# Page Type Router
# 页面类型路由规则

## 1. 路由目标

根据原始 PPT / 文档中的内容结构，判断每页应使用哪种页面类型与版式变体。

页面类型来自大会 PPT 线稿抽象，不按每一页硬拆，而是归纳为可复用结构模式。

## 2. 默认页面类型池

### 2.1 CoverPage 封面页

适用内容：

- 大会主题
- 汇报主题
- 演讲人信息
- 品牌开场
- 项目标题

结构：

```text
品牌标识
中心主标题
副标题 / 演讲人信息
背景氛围
```

组件能力：MainTitle、Subtitle、SpeakerInfo、BackgroundVisual。BrandLogo 不默认生成；仅当原始 PPT 正式内容或用户明确要求时才作为独立组件输出。

---

### 2.2 AgendaPage 目录 / 议程页

适用内容：

- 演讲目录
- 章节结构
- 会议议程
- 内容路线

结构：

```text
页面标题
章节序号
章节名称
章节辅助说明
```

组件能力：PageTitle、AgendaList、AgendaItem、SectionNumber。

---

### 2.3 MetricOverviewPage 核心成果总览页

适用内容：

- 核心指标
- 增长数据
- 业务成果
- 多个关键数字并列展示

结构：

```text
结论标题
2~4 个核心指标
指标解释
趋势符号
辅助说明
```

组件能力：PageTitle、MetricCard、MetricGroup、TrendIcon、SummaryNote。

---

### 2.4 MultiColumnComparisonPage 多列指标对比页

适用内容：

- 多类预算对比
- 多场景对比
- 多策略结果对比
- 多模型能力对比

结构：

```text
页面标题
2~4 个对比列
每列包含标题、说明、指标、结论
底部统一总结
```

变体：ThreeColumn、FourColumn、LeftMainRightSub、CenterFocus。

---

### 2.5 EcosystemRelationshipPage 生态关系 / 双端连接页

适用内容：

- 平台生态
- 广告主与媒体关系
- 联盟连接价值
- 多方协作关系

结构：

```text
左侧角色
中心平台 / 连接节点
右侧角色
连接线 / 流向
底部能力支撑
```

---

### 2.6 StrategyPanoramaPage 战略全景页

适用内容：

- 大盘策略
- 阶段规划
- 多业务布局
- 未来增长方向

结构：

```text
主标题
多个阶段 / 方向模块
关键动作
底部统一结论
```

---

### 2.7 StageEvolutionPage 阶段演进页

适用内容：

- 浅度预算、深度预算、超深度预算
- 过去、现在、未来
- 能力成熟度递进
- 增长阶段变化

结构：

```text
阶段名称
阶段特征
核心指标
阶段状态
演进箭头
```

---

### 2.8 DualCoreArchitecturePage 双核心能力架构页

适用内容：

- 双目标驱动
- 双核心指标
- 两侧能力支撑
- 技术架构与业务目标结合

结构：

```text
左侧能力组
中心核心指标
右侧能力组
底部结论
```

---

### 2.9 FormulaDecompositionPage 公式驱动 / 因素拆解页

适用内容：

- 指标公式
- 影响因素拆解
- CTR × CVR → SCVR
- 多因素共同驱动结果

结构：

```text
核心公式
因素拆解
每个因素的解释
最终结果
```

规则：不得拆散公式关系；运算符号必须保留；因素与结果的方向关系必须保留。

---

### 2.10 CaseStudyPage 重点案例分析页

适用内容：

- 品牌案例
- 项目复盘
- 策略实践
- 业务结果证明

结构：

```text
案例背景
核心问题
解决方案
执行路径
业务结果
```

变体：Split、Process、Growth、LeftTextRightChart。

---

### 2.11 ProcessFlowPage 流程链路页

适用内容：

- 转化路径
- 产品链路
- 模型训练链路
- 投放优化链路
- 从输入到输出的过程说明

结构：

```text
起点
多个流程节点
连接箭头
终点结果
```

规则：必须保持原始流程顺序；不得交换节点前后关系；不得省略中间步骤。

---

### 2.12 CentralModelPage 中心对象模型页

适用内容：

- 人群模型
- 用户分层
- 中心策略对象
- 中心能力与外围因素关系

结构：

```text
中心对象
外围能力 / 因素
输入信息
输出结果
```

---

### 2.13 ShowcaseGalleryPage 素材画廊页

适用内容：

- AIGC 素材展示
- 样式展示
- 产品案例展示
- 创意样例陈列

结构：

```text
页面标题
中心素材画廊
左右关键指标
分类标签
```

规则：素材内容不得被误判为装饰；素材说明与指标关系必须保留。

---

### 2.14 TransformationPage 转型跃迁页

适用内容：

- 从旧模式到新模式
- 从浅度到深度
- 能力升级带来的增长
- 两侧状态对比

结构：

```text
左侧起点
中间升级动作
右侧目标状态
方向箭头
结果指标
```

---

### 2.15 CapabilityRoadmapPage 路线图 / 增长曲线页

适用内容：

- 能力建设路线
- 产品演进
- 技术升级路径
- 下一阶段规划

结构：

```text
时间 / 阶段路径
关键节点
节点说明
未来方向
```

变体：Path、Sequence、Curve。

---

### 2.16 CapabilityMatrixPage 能力矩阵页

适用内容：

- 多业务能力
- 多场景支柱
- 产品能力全景
- 四列能力分类

结构：

```text
页面标题
3~5 个能力列
每列包含分类标题、能力项、图标
底部统一行动或总结
```

---

### 2.17 GeneralStructuredPage 兜底结构页

当页面类型无法确定时使用。

规则：

- 必须保留全部正式内容
- 输出无法判断的原因
- 尽量使用标题 + 分组卡片 + 说明的通用结构
- 标记为需要人工确认

## 3. 路由流程

```text
读取页面内容
↓
识别内容主题
↓
识别内容结构
↓
识别信息关系
↓
匹配页面类型
↓
判断是否需要拆页
↓
选择页面变体
```

## 4. 页面类型判断依据

- 是否是主题开场
- 是否包含目录结构
- 是否以数字结果为主
- 是否存在多列对比
- 是否存在阶段递进
- 是否存在生态关系
- 是否存在公式拆解
- 是否存在案例背景、方法、结果
- 是否存在流程链路
- 是否存在中心模型
- 是否以素材展示为主
- 是否存在路线图或未来规划
- 是否存在多能力分类

## 5. 页面类型冲突处理

如果一页同时符合多个页面类型，优先级如下：

```text
封面页
目录页
流程链路页
公式拆解页
案例分析页
阶段演进页
多列指标对比页
核心成果总览页
能力矩阵页
普通策略页
```

无法判断时使用 `GeneralStructuredPage`。

## 封尾页 `ClosingPage`

适用内容：

- 感谢观看
- 谢谢
- Q&A
- 联系方式
- 合作共建
- 开放合作
- 结束语
- 品牌收束

页面角色：

```text
pageRole: closing
backgroundVariant: closing
```

结构：

```text
中心结束语 / 合作主题
可选副标题
可选联系方式 / 二维码 / Logo
封尾背景氛围
```

规则：

- 使用视觉风格 Skill 中的封尾背景
- 重要文案必须是 HTML 可编辑文本
- 二维码、Logo、联系方式如需替换，应作为独立组件
- 不得将可编辑信息烘焙进背景图
