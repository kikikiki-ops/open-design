# Quality Check Rules
# 生成后质量检查清单

## 1. 内容完整性检查

生成后必须检查：

```text
[ ] 正式标题是否全部出现
[ ] 正文是否全部出现
[ ] 数字是否全部一致
[ ] 单位是否全部一致
[ ] 百分比是否全部一致
[ ] 趋势符号是否全部一致
[ ] 标签是否遗漏
[ ] 图表含义是否保留
[ ] 流程顺序是否正确
[ ] 对比关系是否正确
[ ] 因果关系是否正确
[ ] 是否没有添加未经提供的信息
[ ] 是否没有误把编辑备注放入正文
```

## 2. 页面结构检查

```text
[ ] 页面类型是否匹配内容
[ ] 组件选择是否合理
[ ] 内容是否在组件容量范围内
[ ] 是否需要拆页
[ ] 是否存在信息拥挤
[ ] 是否存在层级混乱
[ ] 是否保留原始信息关系
```

## 3. HTML 检查

```text
[ ] 页面尺寸是否正确
[ ] 是否为真实 HTML 文本
[ ] 是否可编辑
[ ] 是否有清晰 DOM 结构
[ ] 是否存在文本溢出
[ ] 是否存在内容被遮挡
[ ] 是否存在无法编辑的整页图片
[ ] data-source-id 是否正确绑定
```

## 4. 视觉风格检查

```text
[ ] 是否继承指定风格 Skill
[ ] 是否符合 3696 × 1008 超宽屏构图
[ ] 是否保持大屏可读性
[ ] 是否避免普通 16:9 拉宽感
[ ] 是否保持低信息密度
[ ] 是否避免赛博炫光、复杂 HUD、厚重阴影
```

## 5. 失败处理

### 5.1 内容检查失败

如果发现正式内容遗漏：

1. 不得交付最终页面
2. 标记遗漏内容
3. 重新生成或补入
4. 必要时拆页

### 5.2 页面溢出

如果内容溢出：

1. 优先调整布局
2. 其次更换页面变体
3. 再次拆页
4. 禁止缩小字号硬塞
5. 禁止删除内容

### 5.3 页面类型不确定

如果页面类型无法确定：

1. 使用 `GeneralStructuredPage`
2. 保留全部内容
3. 输出不确定原因
4. 建议人工确认

### 5.4 编辑备注不确定

如果不确定某内容是否为备注：

1. 不直接进入最终正文
2. 放入 `uncertain_content_inventory`
3. 在检查结果中提示用户确认

## 背景路由检查

```text
[ ] 每页是否输出 pageRole
[ ] 每页是否输出 backgroundVariant
[ ] CoverPage 是否使用 cover 背景
[ ] 正文页是否使用 content 背景
[ ] ClosingPage 是否使用 closing 背景
[ ] 背景图是否没有承载正式正文、数字、单位或图表标签
[ ] 背景层是否不影响正文可编辑性
```

## 常驻品牌信息检查

```text
[ ] 是否没有生成第二个“快手联盟”
[ ] 是否没有重复叠加 Logo
[ ] 是否没有把背景图中的品牌标识再次作为 HTML 文本输出
[ ] 所有品牌文字是否都能追溯到 content_inventory 或用户明确要求
```

## 独立 Logo 检查

```text
[ ] 背景图是否无 Logo
[ ] 是否启用了 FixedBrandLogo 独立组件
[ ] 封面 / 内容 / 封尾是否都显示 Logo
[ ] Logo 是否使用 SVG 或高清透明 PNG
[ ] 是否没有重复生成“快手联盟”文字
[ ] 是否没有从背景图裁切模糊 Logo
```

## 输入诊断检查

```text
[ ] 是否先判断了每页 sourceType
[ ] 图片型页面是否完成 OCR / 视觉解析
[ ] 是否建立 content_inventory
[ ] OCR 内容是否带 confidence
[ ] 低置信内容是否进入 uncertain_content_inventory
[ ] contentInventoryStatus 是否为 ready 或 review_required
[ ] blocked 页面是否被禁止直接生成 HTML
```
