# Placeholder and Edit Instruction Rules
# 编辑备注 / 占位符识别规则

## 1. 用户确认规则

用户已确认：

> 明显编辑指令 / 待修改占位符不作为最终页面正文展示。

因此，编辑备注不进入最终 HTML 正文，但必须进入 `edit_instruction_inventory`，用于人工审校。

## 2. 常见编辑备注 / 占位符

以下内容通常视为编辑备注或待修改占位符：

- `xxx`
- `xx负责人`
- `+pp`
- `ab`
- `TODO`
- `待补充`
- `占位`
- `这里改一下`
- `改成...`
- `替换为...`
- `深度预算改成 ab`
- `去年 AIGC 大盘...改成行业优化`
- 明显不是正式演示文案的内部修改说明

## 3. edit_instruction_inventory

每条识别为编辑备注的内容必须记录：

```json
{
  "id": "edit-001",
  "page": 1,
  "value": "深度预算改成 ab",
  "reason": "疑似内部修改指令",
  "action": "不进入最终页面正文"
}
```

## 4. 不确定内容处理

如果某段内容既可能是正式内容，也可能是备注，必须进入 `uncertain_content_inventory`。

处理规则：

1. 不擅自删除
2. 不直接作为最终正文展示
3. 在输出检查结果中标记“需人工确认”
4. 如果用户确认其为正式内容，再进入 `content_inventory`

## 5. 禁止行为

- 不得把明显编辑备注直接放进最终正文
- 不得完全丢弃编辑备注而不记录
- 不得擅自用编辑备注改写正式内容，除非用户明确要求
- 不得把 `xxx`、`TODO`、`待补充` 当作正式文案美化输出
