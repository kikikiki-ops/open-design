# Content Preservation Rules
# 内容保真与内容账本规则

## 1. 最高优先级

正式内容不改动、不缺失，是本总控 Skill 的最高优先级。

除非用户明确要求优化、改写、精简或重写，否则所有正式信息必须原样保留。

## 2. 必须保留的正式内容

- 原始标题
- 原始副标题
- 正文说明
- 数字
- 百分比
- 单位
- 趋势符号，例如 ↑、↓、+、-、▲
- 标签
- 图表标题
- 图表结论
- 图表数据含义
- 流程节点
- 阶段名称
- 对比关系
- 因果关系
- 上下级关系
- 策略与结果的对应关系

## 3. 默认禁止行为

- 删除正式内容
- 改写正式内容
- 总结替代正式内容
- 修改数字
- 修改单位
- 修改百分比
- 修改趋势方向
- 修改专有名词
- 擅自补充新信息
- 擅自纠正疑似错误
- 合并不同观点
- 改变原始逻辑关系
- 将一个指标拆成无关联的数字与说明

## 4. 允许调整的内容

允许调整：

- 信息位置
- 版式结构
- 页面类型
- 组件组合方式
- 视觉层级
- 分页方式
- HTML DOM 结构

不允许调整：

- 正式文本本身
- 正式数字本身
- 单位与符号
- 信息逻辑关系

## 5. 内容账本 content_inventory

生成页面之前，必须先建立 `content_inventory`。

每条正式内容必须记录：

```json
{
  "id": "source-001",
  "page": 1,
  "type": "title | subtitle | body | metric | label | chart | relation | flow_node | note",
  "value": "原始内容",
  "unit": "单位，如有",
  "trend": "趋势符号，如有",
  "parentId": "上级内容 id，如有",
  "relation": "与其他内容的关系，如有",
  "status": "official"
}
```

## 6. HTML 绑定规则

正式内容进入 HTML 后，应尽量通过 `data-source-id` 绑定来源。

示例：

```html
<h1 data-source-id="source-001">联盟预算：稳健步入深转时代、布局新增长</h1>
<span data-source-id="source-002">3,580 万/天</span>
```

## 7. 生成后检查

生成完成后，必须将 `content_inventory` 与最终 HTML 页面比对。

检查项：

- 每条 official 内容是否出现
- 数字是否一致
- 单位是否一致
- 百分比是否一致
- 趋势符号是否一致
- 上下级关系是否一致
- 流程顺序是否一致
- 对比关系是否一致
- 因果关系是否一致
- 是否误删任何正式内容
- 是否新增未经提供的信息

## 8. 内容过多时

如果单页无法容纳全部正式内容，必须拆页，不得删除信息。

处理顺序：

```text
优化分组
↓
更换页面变体
↓
拆成连续页面
↓
重新核对 content_inventory
```

禁止：

- 缩小字号硬塞
- 用省略号替代
- 删除辅助信息
- 隐藏内容
- 转成图片
