# Test Plan
# 草稿 PPT 测试流程

## 1. 推荐测试策略

不要一开始只测总包，也不要只分开测。推荐两阶段：

```text
第一轮：分开测，定位问题
第二轮：一起测，验证真实效果
最终上线标准：必须通过总包端到端测试
```

## 2. 总控 Skill 单测

输入草稿 PPT，让总控只输出：

- 页面内容账本
- 页面类型判断
- 是否拆页
- 推荐组件能力
- 编辑备注识别结果
- 内容保真检查结果

重点检查：

- 页面类型是否判断正确
- 正式内容是否完整提取
- 编辑备注是否被排除
- 内容关系是否保留
- 是否错误拆页

## 3. 组件 Skill 单测

使用结构化数据测试组件，不直接喂完整 PPT。

重点检查：

- 组件容量是否合理
- 文字是否溢出
- 数字和单位是否完整展示
- 布局是否可编辑
- 组件 props 是否清晰

## 4. 风格 Skill 单测

给同一个简单页面结构，让风格 Skill 渲染。

重点检查：

- 是否符合 3696 × 1008
- 是否是浅暖白背景
- 是否有青绿 + 香槟金流线
- 是否低信息密度
- 是否符合目标视觉方向
- 是否避免赛博炫光和复杂 HUD

## 5. 小范围集成测试

先从草稿 PPT 中挑 3～5 页代表页：

- 封面页
- 数据总览页
- 多列指标页
- 案例分析页
- 路线图页

重点检查：

- 页面类型是否选对
- 组件是否匹配
- 风格是否统一
- 内容是否缺失
- 页面是否可编辑
- 视觉是否达到预期

## 6. 整套 PPT 端到端测试

最后再跑整套。

重点检查：

- 整套页面是否统一
- 章节节奏是否合理
- 页面类型是否重复过多
- 内容是否全部保留
- 是否有页面过挤
- 是否有页面需要拆分
- 是否有组件缺口
